## Epic C-RBAC — RBAC foundation (P0)

Goal: project-role permission matrix exists at all three layers and is the sole source of truth for project-scoped authorization.

### C-RBAC-01 — Register 12 canonical project role UUIDs
- **Scope:** `src/types/rbac.ts::SYSTEM_ROLE_IDS`
- **Migration:** `00149_seed_project_role_uuids.sql` — insert rows into `roles` with IDs `…301..312` matching enum slugs and `role_scope='project'`.
- **SSOT:** `roles` table is sole registry; TypeScript imports at build time via codegen.
- **RBAC:** N/A (registry only).
- **State machine:** none.
- **Seed:** 12 rows.
- **UI:** none.
- **Tests:** `__tests__/rbac/role-registry.test.ts` asserts all 12 slugs resolve to UUIDs and vice versa.
- **Acceptance:** `SYSTEM_ROLE_IDS.PROJECT_EXECUTIVE === '00000000-0000-0000-0000-000000000301'` and 11 analogous assertions pass.

### C-RBAC-02 — DEFAULT_PROJECT_PERMISSIONS matrix
- **Scope:** `src/lib/permissions.ts`
- **Migration:** `00150_project_permission_bundles.sql` — `role_permission_bundles(role, resource, action, allow)` populated per matrix in `01-role-inventory-matrix.md`.
- **SSOT:** DB is canonical; TS const is generated by `scripts/generate-rbac.ts` from `role_permission_bundles`.
- **RBAC:** N/A here; consumed by C-RBAC-03.
- **Seed:** bundles per role class with inheritance expanded.
- **Tests:** snapshot of resolved bundle per role.
- **Acceptance:** every `{role, resource, action}` tuple referenced in `01-role-inventory-matrix.md` resolves to `true`; unlisted tuples resolve to `false`.

### C-RBAC-03 — Wire project role into `requirePermission()`
- **Scope:** `src/lib/api/permission-guard.ts:73-179`
- **Change:** `resolveUserMembership()` now returns both `platformRole` and `projectRole` (if route is project-scoped). `checkPermission()` consults project bundle first; falls back to platform bundle; denies otherwise.
- **Migration:** none (app-layer).
- **Tests:** unit tests per resource×action×role grid (12 roles × 15 representative resources = 180).
- **Acceptance:** a request with project_role=`guest` on resource=`invoices` action=`read` returns 403 regardless of platform role.

### C-RBAC-04 — Unify action vocabulary
- **Scope:** `src/lib/api/permission-guard.ts:117-121`, `src/lib/permissions.ts`, `src/types/rbac.ts:26-29`
- **Change:** single `PermissionAction` union `'create'|'read'|'update'|'delete'|'manage'|'invite'|'approve'|'export'|'configure'|'bulk_invite'|'impersonate'`. Deprecate `view/edit`. Add runtime-fail warning on use.
- **Tests:** lint rule forbidding `'view'`/`'edit'` in new permission calls.
- **Acceptance:** grep for `'view'` and `'edit'` in `src/app/api/` returns zero matches.

### C-RBAC-05 — MFA per bundle
- **Migration:** `00151_mfa_bundle_enforcement.sql` — `auth_settings` gains `mfa_required_bundles jsonb`.
- **Middleware:** `src/middleware.ts` rejects requests targeting privileged bundles without elevated-MFA claim.
- **Acceptance:** settlement/closeout requests without fresh MFA return 403 with `error=mfa_required`.

### C-RBAC-06 — Press pool RLS
- **Migration:** `00152_press_pool_rls.sql` — RLS policy on `zone_access_grants` allowing SELECT for `holder_role='press'` only on zones flagged `is_press_pool=true`.
- **Depends on:** C-CRED-01, C-CRED-02, C-DATA-06.

### C-RBAC-07 — JWT claim cache for project role
- **Middleware:** attach resolved `{project_id, project_role}` to session cookie on role change.
- **Invalidation:** `role_change` DB trigger publishes LISTEN/NOTIFY; API route `/api/auth/refresh-claims` rebuilds claims.

## Epic C-STATE — Role lifecycle state machine (P0/P1)

Goal: every state transition is explicit, allowed transitions are DB-enforced, closeout is immutable, all transitions are auditable.

### C-STATE-01 — `role_stage_transitions` allowlist + trigger
- **Migration:** `00153_role_stage_transitions.sql`
  - `role_stage_transitions(from_state role_lifecycle_state, to_state role_lifecycle_state, allowed_role project_role[])` primary key `(from_state,to_state)`.
  - `fn_enforce_lifecycle_transition()` BEFORE UPDATE trigger on every table with `lifecycle_state`.
- **Seed:** 32 canonical transitions × role eligibility (see below).
- **Acceptance:** attempt to set `lifecycle_state` from `discovery` directly to `deployment` is rejected with SQLSTATE `22000` and message `invalid_role_lifecycle_transition`.

### C-STATE-02 — Closeout immutability trigger
- **Migration:** `00154_closeout_immutability.sql` — trigger rejects UPDATE/DELETE when `OLD.lifecycle_state = 'closeout'` unless caller has platform role = `developer` AND provides `app.closeout_override_reason` session var.
- **Acceptance:** any UPDATE to a closed record from a non-developer session returns `closeout_immutable`.

### C-STATE-03 — Lifecycle column on `project_users`
- **Migration:** `00155_project_users_lifecycle.sql` — add `lifecycle_state` + backfill from child tables if present, else `discovery`.
- **SSOT:** `project_users` becomes canonical for role lifecycle; `advance_collaborators` and `project_collaborators` keep theirs as denormalized caches updated via trigger.

### C-STATE-04 — `project_role_events` audit table
- **Migration:** `00156_project_role_events.sql`
  ```sql
  CREATE TABLE project_role_events (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id uuid NOT NULL REFERENCES projects(id),
    user_id uuid NOT NULL,
    role project_role NOT NULL,
    from_state role_lifecycle_state,
    to_state role_lifecycle_state NOT NULL,
    actor_id uuid NOT NULL,
    actor_role text NOT NULL,
    reason text,
    at timestamptz NOT NULL DEFAULT now()
  );
  ```
- AFTER UPDATE trigger on `project_users.lifecycle_state` writes event.

### C-STATE-05 — Project closeout gate
- **Migration:** `00157_project_closeout_gate.sql` — `fn_project_closeout_eligibility(project_id)` returns true iff every `project_users` row has `lifecycle_state IN ('archival','closeout')`. BEFORE UPDATE on `projects.status='closed'` rejects otherwise.

### C-STATE-06 — Preconditions table
- **Migration:** `00158_role_stage_preconditions.sql`
  ```sql
  CREATE TABLE role_stage_preconditions (
    role project_role NOT NULL,
    stage role_lifecycle_state NOT NULL,
    check_kind text NOT NULL,          -- e.g. 'coi_valid'
    required bool NOT NULL DEFAULT true,
    PRIMARY KEY (role, stage, check_kind)
  );
  ```
- `fn_check_preconditions(project_user_id, target_state)` returns list of failed checks.
- Trigger from C-STATE-01 calls this before allowing transition.

## Epic C-CRED — Credentials + zone access (P0)

### C-CRED-01 — `credentials` table
- **Migration:** `00159_credentials.sql`
  ```sql
  CREATE TYPE credential_tier AS ENUM ('all_access','production','vendor','talent','press','guest','general_admission');
  CREATE TABLE credentials (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id uuid NOT NULL REFERENCES projects(id),
    holder_user_id uuid NOT NULL,
    holder_role project_role NOT NULL,
    tier credential_tier NOT NULL,
    qr_token text NOT NULL UNIQUE,
    issued_by uuid NOT NULL,
    issued_at timestamptz NOT NULL DEFAULT now(),
    revoked_at timestamptz,
    revoked_reason text
  );
  ```
- RLS: read per project; write `holder_role IN ('management','production','executive')`.

### C-CRED-02 — `zone_access_grants` + `zone_access_events`
- **Migration:** `00160_zone_access.sql`

- Trigger: INSERT to `zone_access_events` checks active grant + active credential + no conflicting capacity.
- Index: `(credential_id, scanned_at)`.
